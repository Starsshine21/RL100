## 🎯 核心算法实现

### 1. IQL评价网络（期望回归）

```python
# V网络：τ=0.7期望回归
L_V = expectile_loss(V(s), Q(s, a_data), τ=0.7)

# Q网络：带目标V的Bellman备份
L_Q = MSE(Q(s,a), r + γV_target(s'))

# 优势：在所有K个PPO步骤中共享
A = Q(s, a) - V(s)
```

**原理**：
- τ=0.7使V(s)偏向Q值的70%分位数
- 这产生了乐观的价值估计
- 避免了显式max操作的不稳定性

---

### 2. K步子MDP的PPO损失

```python
# 将扩散策略分解为K个子决策
π_θ(a|s) = ∏_{k=1}^{K} π_θ(a_{k-1} | a_k, s)

# 每一步都是高斯分布，均值来自DDIM，方差被裁剪
π_θ(a_{k-1}|a_k, s) = N(μ_DDIM, σ_k²), σ_k ∈ [0.01, 0.8]

# PPO损失在K步上求和
L_PPO = Σ_{k=1}^{K} min(r_k · A, clip(r_k, 1-ε, 1+ε) · A)
```

**核心创新**：
- 传统扩散策略难以计算似然π_θ(a|s)
- RL-100将其分解为K个易于计算的高斯步骤
- 每步的对数概率可以直接计算
- 使用共享的优势A作为学习信号

---

### 3. 一致性蒸馏

```python
# 教师：K步DDIM去噪
a_teacher = DP3.conditional_sample(noise, obs, K=10)

# 学生：1步直接预测
a_student = ConsistencyModel(noise, obs)

# 蒸馏损失
L_CD = ||a_student - a_teacher||²
```

**好处**：
- 推理速度提升10倍（1步 vs 10步）
- 保持K步生成的质量
- 可以在RL训练过程中更新

## 📐 训练流程

### 阶段1：模仿学习（1000轮）

```
对于每个epoch：
    对于每个batch：
        1. 编码观察 → obs_features
        2. 采样时间步 t ~ U(0, T)
        3. 添加噪声：a_t = √α_t · a + √(1-α_t) · ε
        4. 预测：ε_pred = UNet(a_t, t, obs_features)
        5. 损失：L_BC = ||ε_pred - ε||²
        6. 反向传播并优化
        7. 更新EMA

    每100个epoch：
        - 在环境中评估
        - 记录成功率到WandB
```

**输出**：训练好的扩散策略π_θ

---

### 阶段2：离线RL循环（5次迭代）

**对于每次迭代i：**

**2a. 训练IQL评价网络（50轮）**

```
对于每个epoch：
    对于每个batch：
        # 获取数据
        s, a, r, s', done = batch

        # 使用冻结的DP3编码器编码
        obs_features = DP3.obs_encoder(s)

        # 更新V网络
        v_loss = expectile_loss(V(obs_features), Q(obs_features, a), τ=0.7)
        v_optimizer.step()

        # 更新Q网络
        q_target = r + γ * V_target(next_obs_features)
        q_loss = MSE(Q(obs_features, a), q_target)
        q_optimizer.step()

        # 软更新目标V
        V_target ← 0.005 * V + 0.995 * V_target
```

**2b. 使用PPO优化策略（100轮）**

```
对于每个epoch：
    对于每个batch：
        # 计算优势
        A = Q(obs_features, a) - V(obs_features)

        # 使用旧策略采样轨迹
        a_final, [a_T, ..., a_0], [log_p_0, ..., log_p_{K-1}] =
            policy.sample_with_trajectory(s)

        # 使用新策略重新计算对数概率
        [log_p_0', ..., log_p_{K-1}'] = recompute_log_probs([a_T, ..., a_0], s)

        # 在K步上计算PPO损失
        ppo_loss = Σ_{k=0}^{K-1} -min(r_k * A, clip(r_k) * A)

        # 优化
        policy_optimizer.step()

        # 一致性蒸馏（每5步）
        if step % 5 == 0:
            cd_loss = MSE(student(noise, s), teacher_K_step(noise, s))
            consistency_optimizer.step()
```

**2c. 收集新数据（50个回合）**

```
对于每个回合：
    重置环境
    使用EMA策略执行rollout
    收集轨迹：(s, a, r, s', done)
    添加到D_new
```

**2d. 重新训练模仿学习（100轮）**

```
合并数据集：D ← D ∪ D_new
在合并的数据集上重新训练模仿学习
```

## 🎓 关键数学洞察

### 为什么K步子MDP有效

**问题**：扩散策略的似然π_θ(a|s)难以计算

**解决方案**：分解为K个易于计算的高斯步骤
```
π_θ(a|s) = ∏_{k=1}^{K} N(a_{k-1}; μ_k, σ_k²)
```

每一步的对数概率容易计算：
```
log p(a_{k-1}|a_k, s) = -½[d·log(2πσ_k²) + ||a_{k-1}-μ_k||²/σ_k²]
```

**结果**：现在可以计算策略梯度并使用PPO！

---

### 为什么V使用期望回归（τ=0.7）

**目标**：V(s)应该估计max_a Q(s,a)

**为什么不直接取max**：在离散数据集上不稳定

**期望解决方案**：将V拟合到Q(s, a_data)的70%分位数
```
L_V = E[ |τ - 𝟙{Q < V}| · (V - Q)² ]
```

**效果**：
- 高估（V > Q）权重0.7 → 惩罚更多
- 低估（V < Q）权重0.3 → 惩罚更少
- 净效应：V被推向上分位数

---

### 为什么需要一致性蒸馏

**问题**：K步去噪很慢（10次网络调用）

**解决方案**：训练学生在1步内匹配教师输出
```
L_CD = ||CM(noise) - DDIM^K(noise)||²
```

**结果**：推理速度提升10倍，质量相同！
