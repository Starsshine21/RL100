# RL100 进一步修复说明

日期：2026-03-13  
对应审计文档：`RL100_进一步实现审计_20260313.md`

---

## 本轮修复内容

本轮按进一步审计报告，实际落地了 3 组修改：

1. 去掉训练主路径里对 `EMA policy` 的默认依赖  
2. 修正 `cd_every` 在 PPO inner-loop 里的实际触发语义  
3. 缓存 PPO 路径中的 observation encoding，减少重复前向

另外补了一项工程提示：

4. 当作业可见多张 GPU 但训练仍是单卡时，启动日志会明确告警

---

## 1. OPE / rollout / online collection 默认改回当前 policy

### 修改

- `3D-Diffusion-Policy/diffusion_policy_3d/rl100_trainer.py`
  - `get_runtime_policy(..., use_ema=False)` 默认改为不用 EMA
  - `_evaluate_policy_amq()` 现在固定评估 `self.policy`
  - `collect_new_data()` 新增 `use_ema` 参数，并从 runtime 配置读取 `collection_use_ema`
  - offline collection 和 online collection 都显式传入 `use_ema=False`

- `3D-Diffusion-Policy/train_rl100.py`
  - final evaluation 新增 `runtime.final_eval_use_ema`

- `3D-Diffusion-Policy/eval_rl100.py`
  - checkpoint 评测新增 `runtime.eval_use_ema`
  - 默认加载 `policy`，只有显式要求时才加载 `ema_policy`

### 结果

现在训练主路径中的：

- OPE gate
- offline 数据扩展
- online 数据收集

默认都和当前正在优化的 `π_θ` 对齐，不再默认切到 EMA policy。

---

## 2. `cd_every` 现在按 outer batch 触发一次，不再在同一 batch 里连跑 4 次

### 修改

- `3D-Diffusion-Policy/diffusion_policy_3d/rl100_trainer.py`
  - offline PPO 中，先在 outer batch 级别计算 `run_cd_this_batch`
  - 只有 `ppo_inner == 0` 时才允许本 batch 触发 consistency distillation
  - online PPO 也同步改成每个 batch 最多触发 1 次 CD

### 结果

`cd_every=5` 现在才真正接近“每 5 个 batch 做一次 consistency distillation”的配置语义。  
不会再出现：

- 同一个 batch 命中一次 CD 条件
- 却在 4 个 inner PPO step 里连做 4 次 CD backward / optimizer step

这会直接减少 PPO 阶段的额外计算量。

---

## 3. PPO 路径加了 observation conditioning cache

### 修改

- `3D-Diffusion-Policy/diffusion_policy_3d/rl100_trainer.py`
  - 新增 `_encode_obs_representations()`
  - 单次 encoder 前向同时返回：
    - critics 用的 flatten `obs_features`
    - diffusion UNet 用的 `global_cond`

- `3D-Diffusion-Policy/diffusion_policy_3d/policy/rl100_policy.py`
  - 新增 `encode_obs_global_cond()`
  - `sample_for_ppo()` 支持直接接收缓存好的 `global_cond`
  - `compute_ppo_loss()` 支持直接接收缓存好的 `global_cond`

- `3D-Diffusion-Policy/diffusion_policy_3d/model/rl/consistency_model.py`
  - `compute_distillation_loss()` 支持接收缓存好的 `global_cond`
  - 当外部传入缓存条件时，会 `detach()`，保持 consistency distillation 不去反向更新 obs encoder 的既有语义

### 结果

offline PPO 里同一个 batch 不再重复做这些工作：

- normalize obs
- point cloud 裁剪
- obs encoder 前向
- reshape 成 `global_cond`

而是一次编码后复用到：

- advantage 计算
- `sample_for_ppo()`
- `compute_ppo_loss()`
- consistency distillation

这会明显降低 PPO 阶段的重复前向开销。

## 4. 增加“4 卡作业但当前训练仍单卡”的显式告警

### 修改

- `3D-Diffusion-Policy/train_rl100.py`
  - 若 `torch.cuda.device_count() > 1`，启动时打印告警：
    当前 RL100 trainer 仍是单设备训练

### 结果

你之后再申请多卡作业时，日志会直接告诉你当前代码是否真的在多卡跑，避免误判。

## 本轮未做的事

本轮**没有**直接把训练改成真正的 4 卡 DDP。

原因是这已经不是局部逻辑修补，而是一次训练框架级改造，至少会涉及：

- launcher
- rank / local_rank
- DistributedSampler
- checkpoint/save/load
- EMA / env runner / rollout 行为在多进程下的职责划分

这可以继续做，但建议作为下一步单独实施，而不是和这轮语义修补混在一起。
